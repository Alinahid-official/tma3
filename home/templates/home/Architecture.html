Architecture

The image above shows a high level overview for the Architecture of the assignment. As the image shows, the assignment is run on a Django Web Framework. Essentially, the entire website is being run on a live VM hosted by digital ocean. On the VM itself, there are several services installed, including Django, NGINX, as well as SqlLite. I chose chose Django and this architecture because Django is a very relevant webframework in todays world, and digital ocean droplets are free for smallscale project such as this one. I was told in the COMP 466 discussion forums Django was an acceptable framework for this assignment, and I had a lot of fun learning it

Starting from the left side of the diagram, the client (you) will connect to the web server via http through the ip address of the server. All the user needs to do is enter the URL http://143.198.71.129:8000/ in their webrowser in order to access this web application. The entire application is being run on port 8000. This webserver is used for various other projects of mine, so you must specify the port in the url when you first connect. Note, all browser testing for this assignment was done using google chrome and firefox. I have not tested compatability on other browsers. I did not want to set a domain name for this website being that it is a smallscale website used only for the purposes for this course, so for now please connect using IP.

This entire assignment, is one Django project, and 5 Django apps. This single Django project will contain apps for each part of this assignment. There is the documentation/homepage app, the checkVisit app, the slideShow app, the part3 app, and part4 app. The project submission folder that was submitted to myAU contains the project root folder SimpleDev, which is the name of this project. Inside this root folder, there is a folder for each app, a SimpleDev project settings folder, and the database for this assignment. By default, Django uses SQLite for its DB. Every Django project when created comes with a default db. I chose to stick with this db rather then migrate to a larger db platform such as MySql because this project is relatively small scale, and has no need to use a different db platform. The db is used primarily for part 4 when it is required. In order to follow Djangos reccomended project structure, I unfortunately had to disregard the file structure that was required for this assignment, so that is why you will see a different file structure in the submission for this assignment then what is required.

Each app folder (home, checkVisit, slideShow, part3, and part4) contains folders related to each app. When you the client wishes to connect to an app, Django will look insde the related app folder to fetch the relevant files required to run the app. Every app folder has a static folder, where static files such as scripts and images are located,a template folder where all the html files for an app are located, a urls.py file where all the urls relevant to the app are contained, and a views folder which tell Django what to do when the you the client sends a url to the server. For instance, if I enter the url http://143.198.71.129:8000/part4/, django will recieve the get request and look in the urls.py file inside the part4 app folder. From there it will look for instructions what to do with that URL. In this case, Django will run the index view, which from there will render the part 4 index html file.

All apps for this project follow this formula. Find the url in the urls.py file, then run the corresponding view in the views.py file to then allow Django to decide what to do. Communication between the client and the web server is all done mostly using javascript and running get requests. All apps will also contain a very similar UI. A navigation bar on top, the content displayed in the middle, and a sidebar to the left. The sidebar is made visible by clicking the SIMPLEDEV logo on the top left of the navigation. It is here that you the client can connect to any app in this assignment you wish.
