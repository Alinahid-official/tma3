The above Image Shows the Architecture of the project. Project si build on Django . It is a modern webframework used to develop high quality websites. Our website is hosted in Amazon Webservices. The operating system used in the aws server is Ubuntu. The application used to create the server are nginx gunicorn and superadvisor. The public ip is http://3.86.255.16/ can be accessed from any where around the globe

Starting from the left side of the diagram, The client can access the page or webiste from browser simply using the link http://3.86.255.16/ then client can navigate through any part the project using the various link in the navigation bars. Whenever any link in clicked the django server then request for the page client is looking for. Then the django server completes all the required services using django SQL DB lite and render the page using django templates . The project is only tested in chorme browser. The domain name is not purchased as this is not a commercial project it is only used for this course.

The Entire assignment is a single Djnago Project with 5 apps inside it. Talking about the root directory consist of 5 app directories and three other directories one for share static files like css imgs and javascript and other one is template folder which is used for common templates used over the different app. The last directory is django engine directory which maps the services of each app to urls I have used Django DB sql lite database which is default database for django as per the requirement, I have only used database for 2nd part and 4th part as mentioned in the assignment.

In each app folder there are by default files by django like models.py, app.py, urls.py, views.py. There are 5 apps as mentioned earlier thus each folder have or app has its own templates , view and urls. When any services is called by the client the django lokks for the required app and its urls whose works are defined the views.py file including all the crud ooperations

All apps works similarly. We have navigation in every part where client can navigate to the any part. Once any app is called it looks for the required service within its folder and response back.
