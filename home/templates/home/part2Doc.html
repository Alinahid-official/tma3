Part 2 Documentation

All the code and files used for this app are contained inside the slideShow folder. To connect to this app, either press the link in the sidebar, or you can directly connect to it by entering the URL http://ocalhost:8000/part2/

Upon connecting to this app, the main2.js script is run (located in the static folder). In this app, there are 20 images located within the static folder. The goal is to fetch these images from the server and render them in a slide show on the client side. To achieve this, we will use blob fetch to get the image data. In main2.js, we first fetch the relevant json data for the images. In the slideShow app folder there is a json file called Image_data. Each entry in this json file contains a caption and a imageURL for each of the images. So, on the client end we first send a get request to the server. This get request has a very special URL in that it is a commmand URL. Within the slideShow app folder, I have created a folder called command. This command folder contains special URLs that the client can run in order to perform certain command. In our case, we will run GET /slideshow/command/getJson/. When this get request is sent, Django will look in the command URL file to look for the corresponding command. It then runs the getJson function in the views. This function will read the json data stored on the server, and send it back to the client in JSON format. With this JSON data, the client now has the caption data and image url data for each of the images.

When you first connect to the slideshow app, it might take around 20-30 seconds to preload all the images. Once all the images are loaded, the buttons will load at the bottom navigation. To start the slideshow, select the sequence order which is located above the canvas, and then press the start button in the bottom button navigation. The images for the slideshow are obtained in a similar fashion to how the json data was obtained. The client runs a command GET request, and the server returns the client the image in blob format.
